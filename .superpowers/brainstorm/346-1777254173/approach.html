Como o mapa deve ser construído?

O requisito "exatamente fiel ao PDF" pode ser atendido de três formas. Cada uma tem implicações diferentes para manutenção e fidelidade visual.

🗺️ IMAGEM DO PDF

Imagem como fundo · pontos de toque invisíveis por cima

A — Imagem do PDF como fundo

Converte o PDF em PNG/WebP, usa como asset de imagem. Camadas invisíveis de toque são desenhadas na frente para detectar linhas e estações.

Prós

  • 100% fiel visualmente
  • Implementação rápida
  • Zoom nativo via InteractiveViewer

Contras

  • Requer conversão do PDF → imagem
  • Filtro por linha é overlay semi-transparente (não isola linhas reais)
  • Tamanho de asset maior

Redesenho vetorial preciso · coordenadas mapeadas do PDF

B — Redesenho vetorial fiel

Mapeamento manual de todas as coordenadas do PDF para o canvas atual (2400×1800). Cada segmento de linha e posição de estação reposicionado para corresponder ao PDF.

Prós

  • Filtro por linha perfeitamente isolado
  • Sem asset externo
  • Dark mode funciona naturalmente

Contras

  • Requer medir/mapear ~200+ coordenadas
  • "Exatamente fiel" depende da precisão manual
  • Tempo de implementação maior
L3 Vermelha

Imagem base + linhas vetoriais de destaque por cima

C — Híbrido (imagem + vetor interativo)

Imagem do PDF como base fixa. Quando nenhuma linha está selecionada, mostra a imagem pura. Ao selecionar, sobreposição vetorial destaca a linha escolhida e apaga as outras.

Prós

  • Fidelidade total no modo visão geral
  • Filtro com boa separação visual
  • Implementação moderada

Contras

  • Requer conversão do PDF E coordenadas aproximadas
  • Duas camadas para manter sincronizadas
